| P11 | 업무 지시 | 누구나 가능(조직도 제한 옵션) → 재작업(rework) 과제 생성 |

| P16 | 분석 | 업무 흐름 병목·사이클타임·처리량·재작업률 가시화 |

## 구현 단계 (자동 진행 결정)

| # | 결정 | 선택(권장안) | 대안 | 이유 |
|---|---|---|---|---|
| D1 | 스택 | Next.js 15 App Router + TS + Prisma + PostgreSQL | NestJS+React 분리 | 단일 코드베이스, 빠른 MVP, 멀티테넌시 용이 |
| D2 | 인증 | 자체 JWT(httpOnly 쿠키)+bcrypt | NextAuth | 멀티테넌트 신원(테넌트별 email) 제어가 직접 구현이 단순 |
| D3 | 스타일 | Tailwind CSS v4 | 순수 CSS | 빠르고 일관된 UI |
| D4 | 결재자 해석 | 조직도에서 런타임 해석(상사 체인/전결 금액티어) | 정적 저장 | 인사이동에 강함 |
| D5 | 인스턴스 그래프 | 인스턴스가 자체 노드/엣지 보유(템플릿 복제) | 정의 그래프 직접 참조 | 사후 수정(협조업무 삽입) 위해 인스턴스 독립 필요 |
| D6 | 그래프 에디터 | 폼 기반 노드/엣지 편집 + 자동배치 SVG 순서도 뷰 | 풀 드래그앤드롭 | MVP 안정성. 드래그앤드롭은 후속 |
| D7 | AI 생성 | claude-opus-4-8 + 구조화 출력. 키 없으면 휴리스틱 폴백 | AI 필수 | 키 없이도 앱 동작 보장 |
| D8 | 금액 타입 | Int(원) | BigInt | 직렬화 단순. SMB 범위 충분(~21억) |
| D9 | 업무지시 모델 | 단일 Directive(누구나, 테넌트 설정으로 조직도 제한) → rework | binding/suggestion 분리 | 사용자 선택(제한은 토글) |
| D10 | 협조업무 인라인 삽입 | afterNode→새노드→기존후속 으로 엣지 재배선 | 회고적 재활성화 | 요청자가 해당 노드 진행 중 가정(MVP) |
| D11 | 협조업무 병렬 | 곁가지(본 흐름 비차단), 인스턴스 완료는 END 기준 | 모든 노드 종결 대기 | 단순/예측가능. 차단형은 INLINE로 |
| D12 | 결재자 자동승인 | 결재선이 비면(상사 없음) 자동 승인 | 차단 | SMB 최상위(대표) 기안 시 막히지 않게 |
